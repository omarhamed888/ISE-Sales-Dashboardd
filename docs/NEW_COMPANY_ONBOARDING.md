# New Company Onboarding Runbook

Steps to spin up this sales dashboard for a new company.
Estimated time: ~20 minutes.

---

## Prerequisites

- Node.js 18+ and `npm` installed
- Firebase CLI logged in: `firebase login`
- `tsx` available: `npm install -g tsx`
- Access to Google Cloud / Firebase console

---

## Step 1 — Create Firebase Project

1. Go to https://console.firebase.google.com
2. **Add project** → name it e.g. `acme-sales`
3. Disable Google Analytics (not needed) → Create project

**Enable services:**
- **Firestore Database** → Create database → Production mode → Region: `eur3` (EU) or `nam5` (US)
- **Authentication** → Get started → Email/Password → Enable
- **Hosting** → Get started (click through the wizard, no need to install CLI again)

**Get credentials:**
- Project Settings → Your apps → Add app → Web (`</>`)
- Register app (nickname e.g. "Sales Dashboard") → Copy the `firebaseConfig` object
- Project Settings → Service accounts → **Generate new private key** → Save as `keys/acme-sa.json`

---

## Step 2 — Create env file

```bash
cp .env.template .env.acme
```

Fill in `.env.acme` with values from Firebase console:

```env
VITE_FIREBASE_API_KEY=AIza...
VITE_FIREBASE_AUTH_DOMAIN=acme-sales.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=acme-sales
VITE_FIREBASE_STORAGE_BUCKET=acme-sales.firebasestorage.app
VITE_FIREBASE_MESSAGING_SENDER_ID=123456789
VITE_FIREBASE_APP_ID=1:123456789:web:abc...

# Company's own Gemini API keys (get from Google AI Studio)
VITE_GEMINI_API_KEY_1=AIza...
VITE_GEMINI_API_KEY_2=AIza...
VITE_GEMINI_API_KEY_3=AIza...
```

---

## Step 3 — Upload company logo

1. Firebase console → Storage → Get started → Start in production mode
2. Upload the company logo file (PNG/SVG, ideally 200×200px)
3. Click the uploaded file → Copy the **Download URL** (starts with `https://firebasestorage...`)

---

## Step 4 — Seed the company

```bash
npx tsx scripts/seed-company.ts \
  --company="Acme Corp" \
  --adminEmail="admin@acme.com" \
  --adminName="Ahmed Mohamed" \
  --logoUrl="https://firebasestorage.googleapis.com/v0/b/acme-sales.appspot.com/o/logo.png?alt=media&token=..." \
  --currency="EGP" \
  --currencyLabel="جنيه" \
  --dataQualityDate="2026-06-01" \
  --serviceAccount="./keys/acme-sa.json"
```

This creates:
- `app_config/settings` document with company branding + config
- The superadmin user in Firebase Auth + `users/{uid}` document

**Send the admin their temporary password** via Firebase console:
Authentication → Users → find the email → Reset password link

---

## Step 5 — Deploy Firestore rules + hosting

```bash
# Deploy just this one company:
cp .env.acme .env.local
npm run build
firebase deploy --project acme-sales --only firestore:rules,hosting

# OR deploy ALL companies at once:
./scripts/deploy-all.sh
```

---

## Step 6 — Post-deploy config (in Firestore console)

Open https://console.firebase.google.com → `acme-sales` project → Firestore → `app_config/settings`

Adjust as needed:
| Field | Description | Example |
|-------|-------------|---------|
| `ignoredAdNames` | Ad names to exclude from analytics | `["عام", "تجريبي"]` |
| `conversionGreenThreshold` | % above which conversion is green | `15` |
| `conversionAmberThreshold` | % above which conversion is amber | `5` |
| `reportDeadlineHour` | Hour (24h) by which reps must submit | `16` |
| `workingDays` | Days 0=Sun,1=Mon...6=Sat | `[0,1,2,3,4,6]` |
| `features.attendance` | Enable/disable attendance tracking | `true` |

---

## Step 7 — Create sales reps

1. Log in as superadmin at the deployed URL
2. Go to **صلاحيات الوصول** (`/access`)
3. Add team members with their emails and roles

---

## Verification Checklist

- [ ] Login page shows the correct company logo and name
- [ ] `/settings` page shows correct company config
- [ ] Submit a test report → appears in Firestore `reports` collection of the NEW project only
- [ ] Check another company's Firebase project → data NOT present (isolation confirmed)
- [ ] Change `app_config/settings.companyName` in Firestore → app updates in real-time (no redeploy)

---

## Env file naming convention

| File | Purpose |
|------|---------|
| `.env.template` | Template — copy this for each new company |
| `.env.acme` | Acme Corp production |
| `.env.acme-staging` | Acme Corp staging (optional) |
| `.env.local` | Auto-generated by deploy scripts — do not edit manually |

> **Security:** Never commit `.env.*` files (except `.env.template`). They contain API keys.
> All `.env.*` are in `.gitignore` except `.env.template`.
